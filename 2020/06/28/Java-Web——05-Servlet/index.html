1.定义

  1. ServletContext是一个接口，表示Servlet上下文对象；

HTTP协议

1.定义

  1. 协议：是指双方或多方相互约定好大家都需要遵守的规则。

  2. HTTP协议：客户端和服务器进行通信时发送的数据需要遵守的规则。

  3. HTTP协议中的数据又叫报文。

2.请求Request的HTTP协议格式

客户端给服务器发送数据叫做请求，请求又分为GET请求和POST请求。

GET请求

  1. 请求行
  • 请求的方式——GET
  • 请求的资源路径——[+?+请求参数]
  • 请求的协议版本号——HTTP/1.1
  1. 请求头
  • key : value——键值对——不同的键值对表示不同的含义
  1. 常见的键值对
名称 说明
Accept 告诉服务器客户端可以接收的数据类型
Accept-Language 告诉服务器客户端可以接收的语言类型，zh_CN—中文、en_US—英语
User-Agent 浏览器的相关信息
Accept-Encoding 告诉服务器客户端可以接收的数据编码格式
Host 表示请求的服务器ip和端口号
Connection 告诉服务器请求连接如何处理，Keep-Alive—服务器回传完数据保持连接一段时间、Closed—回传完数据马上关闭
  1. GET请求有form标签、a标签、link标签引入css、Script标签引入js文件、img标签引入图片、iframe引入html页面、在浏览器地址栏中输入地址后敲回车。

POST请求

  1. 请求行
  • 请求的方式——POST
  • 请求的资源路径——[+?+请求参数]
  • 请求的协议版本号——HTTP/1.1
  1. 请求头
  • key : value——键值对——不同的键值对表示不同的含义

  1. 请求体——发送给服务器的数据

  2. 常见的键值对

名称 说明
Accept 表示客户端可以接收的数据类型
Accept-Language 表示客户端可以接收的语言类型，zh_CN—中文、en_US—英语
Referer 表示请求发起时，浏览器地址栏中的地址
Content-Type 表示发送的数据类型，application/x-www-form-urlencoded：表示提交的数据格式是:name=value&name=value，然后对其进行url编码，url编码是把非英文内容转换为：%xx%xx；multipart/form-data表示以多段的形式提交数据给服务器（以流的形式提交，用于上传）
Content-Length 表示发送的数据的长度
Cache-Control 表示如何控制缓存，no-cache表示不缓存
  1. POST请求有method=post。

常用的请求头

名称 说明
Accept 表示客户端可以接收的数据类型
Accept-Language 表示客户端可以接收的语言类型
User-Agent 表示客户端浏览器的信息
Host 表示请求时的服务器IP和端口号

3.响应Response的HTTP协议格式

服务器给客户端回传数据叫做响应。

  1. 响应行
  • 响应的协议和版本号
  • 响应状态码
  • 响应状态描述符
  1. 响应头
  • key : value——不同的响应头有不同的含义

  1. 响应体——回传给客户端的数据

  2. 常见的键值对

名称 说明
Server 服务器信息
Content-Type 表示响应体的数据类型
Content-Length 响应体的长度
Date 请求的响应时间（格林时间）
  1. 常用的响应码说明
响应码 说明
200 表示请求成功
302 表示请求重定向
404 请求地址错误
500 代码错误

HttpServletRequest类

1.作用

每次只要有请求进入Tomcat服务器，Tomcat服务器都会把请求过来的HTTP协议信息解析好封装到Request对象中，然后传递到Service方法中给我们使用。可以通过HttpServletRequest对象，获取到所有请求的信息。

2.HttpServletRequest类常用的方法

方法 作用
getRequestURI() 获取请求的资源路径
getRequestURL() 获取请求的绝对路径
getRemoteHost() 获取客户端的ip地址
getHeader() 获取请求头
getParameter() 获取请求参数(单个参数值)
getParameterValues() 获取请求参数(多个值)
getMethod() 获取请求的方式GET或POST
setAttribute(key,value) 设置域数据
getAttribute(key) 获取域数据
getRequestDispatcher() 获取请求转发对象

HttpServletResponse类

1.作用

每次请求进来，Tomcat服务器都会创建一个response对象传递给Servlet程序去使用。HttpServletResponse表示所有响应的信息，通过HttpServletResponse对象可以设置返回给客户端的信息。

2.输出流

字节流 getOutputStream(); 常用于下载（传递二进制数据）
字符流 getWriter(); 常用于回传字符串（常用）
  • 两个流只可以同时使用一个，使用了字节流就不能使用字符流。

5.请求重定向
