  1. Cookie生命控制指的是如何管理Cookie在什么时候被销毁。
  2. setMaxAge()：
  • 正数表示在指定秒数后Cookie被销毁；

Cookie有效路径Path的设置

  1. Cookie的path属性可以有效的过滤哪些Cookie可以发送发送给服务器，哪些不发送给服务器。

  2. path属性是通过请求的地址来进行有效的过滤。

  3. 例：

  • Cookie A path=/工程路径
  • Cookie B path=/工程路径/abc。

请求地址如下：

  1. http://ip:port/工程路径/a.html

  • Cookie A 发送

  • Cookie B 不发送

请求地址如下：

  1. http://ip:port/工程路径/abc/a.html

  • Cookie A 发送

  • Cookie B 发送

2.Session会话

定义

Session会话就是一个接口(HttpSession)，是一种用来维护一个客户端和服务器之间关联的技术，每个客户端都有自己的一个Session会话。

作用

经常使用Session会话保存用户登录之后的信息。

创建和获取Session

  1. 创建和获取Session的API相同。
  2. request.getSession()：
  • 第一次调用：创建Session会话；
  • 之后调用：获取前面创建好的Session会话对象。
  1. isNew()：判断Session是否为新创建的Session：
  • true：新创建的Session；
  • false：之前创建的Session。
  1. 每个会话都有唯一的一个ID值：
  • getId()：获取Session会话的ID值。

Session域中数据的存取

Session生命控制

  1. public void setMaxInactiveInterval(int interval)————设置Session的超时时间(以秒为单位)，超过指定的时长，Session就会被销毁。
  • 值为正数：设定Session的超时时长；
  • 值为负数：永不超时。
  1. public int getMaxInactiveInterval()————获取Session的超时时间。
  2. public void invalidate()————让当前Session会话马上超时失效。
  3. setMaxInactiveInterval(int interval)————修改单个Session的超时时长。
  4. Session的默认超时时长为30分钟。
  1. 修改Web工程中所有Session的默认超时时长————在web.xml中进行配置

超时定义

Session的超时指的是客户端两次请求的最大时间间隔。
